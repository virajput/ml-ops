# Heart Disease Risk Prediction — MLOps Pipeline

End-to-end MLOps solution: a reproducible pipeline that cleans the UCI Heart Disease dataset, trains and tracks two classifiers, packages the best
model, serves it behind a FastAPI `/predict` endpoint, and deploys it to Kubernetes with Prometheus/Grafana monitoring.

## Project layout

```
data/                  download_data.py + raw/processed CSVs
src/heart_disease/     data cleaning, EDA, feature pipeline, training, inference
notebooks              python notebooks for EDA and model development
api/                   FastAPI serving app (schemas.py, main.py)
tests/                 pytest suite (data, features, training, predict, API)
models/                persisted joblib pipelines (best_model.joblib)
reports/figures/       EDA plots, confusion matrices, ROC curves, architecture diagram
mlruns/                local MLflow tracking store (generated by training)
k8s/                   Kubernetes Deployment + Service manifests
monitoring/             Prometheus config + Grafana provisioning (datasource + dashboard)
.github/workflows/      CI pipeline (lint, test, train, docker build+smoke test)
screenshots/            evidence captured for the report
Dockerfile, docker-compose.monitoring.yml, requirements.txt, pyproject.toml
```

## 1. Local setup (clean environment)

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
pip install -e .          # installs the `heart_disease` package from src/
```

## 2. Data acquisition & EDA

```bash
python data/download_data.py        # downloads UCI CSV, cleans it, writes data/processed/
python -m heart_disease.eda         # writes plots to reports/figures/
```

## 3. Train models + MLflow tracking

```bash
python -m heart_disease.train
mlflow ui --backend-store-uri ./mlruns --port 5500   # open http://127.0.0.1:5500
```

Produces `models/best_model.joblib`, `models/logistic_regression.joblib`,
`models/random_forest.joblib`, and per-model confusion-matrix/ROC plots in
`reports/figures/`. Every run's params, metrics, and artifacts are logged to MLflow.

## 4. Run tests & lint

```bash
flake8 src api tests data/download_data.py
pytest tests/ -v --cov=src/heart_disease --cov=api
```

## 5. Run the API locally

```bash
uvicorn api.main:app --reload --port 8080
# Swagger UI: http://127.0.0.1:8080/docs
curl -X POST http://127.0.0.1:8080/predict -H "Content-Type: application/json" -d '{
  "age":63,"trestbps":145,"chol":233,"thalach":150,"oldpeak":2.3,
  "sex":1,"cp":1,"fbs":1,"restecg":2,"exang":0,"slope":3,"ca":0,"thal":6
}'
```

## 6. Docker

```bash
docker build -t heart-disease-api:latest .
docker run -d --name heart-disease-api -p 8090:8000 heart-disease-api:latest
curl http://127.0.0.1:8090/health
```

## 7. Monitoring stack (API + Prometheus + Grafana)

```bash
docker compose -f docker-compose.monitoring.yml up -d
# API:        http://127.0.0.1:8090
# Prometheus: http://127.0.0.1:9099
# Grafana:    http://127.0.0.1:3000  (admin/admin, dashboard auto-provisioned)
```

## 8. Kubernetes (Minikube)

```bash
minikube start --driver=docker
eval $(minikube docker-env)          # build the image inside minikube's docker daemon
docker build -t heart-disease-api:latest .
kubectl apply -f k8s/deployment.yaml -f k8s/service.yaml
kubectl get pods,svc
minikube service heart-disease-api-service --url   # prints a local tunnel URL
```

## 9. CI/CD

`.github/workflows/ci.yml` runs on every push/PR to `main`:
1. **lint-and-test** — flake8 + pytest with coverage (fails the pipeline on any error).
2. **train** — downloads data, runs EDA, trains + tracks both models, uploads model/figure
   artifacts.
3. **docker-build-and-smoke-test** — builds the image and verifies `/health` and `/predict`
   respond before the pipeline is considered green.
